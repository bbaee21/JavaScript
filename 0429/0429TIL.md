## JavaScript 문법



### ECMAScript 6 type과 연산자

* 데이터 타입 
  * 크게 `원시(Primitive type)`과 `참조(Reference type) `로 분류된다.
  * **`원시타입`**
    * 객체가 아닌 기본타입 / 변수에 타입의 값이 담긴다 / 복사할 때 실제 값 복사
    * Number(숫자) 타입
      * 부동소수점 형식을 따르며, 정수와 실수 구분 없는 하나의 숫자 타입이다. 
      * `0, -0, NaN`을 제외한 모든 경우 참으로 형변환 된다.
      * ex) Infinity, -infinity (양, 음의 무한대) / NaN(Not-A-Numeber) : 계산 불가능한 경우 반환되는 값
    * String(문자열) 타입
      * 텍스트 데이터
      * `''` `""` 모두 사용 가능
      * Template Literal(템플릿 리터럴)
        * ES6부터 지원된 기능
        * `backtic(``)`  으로 표현
        * ${expression} 가능
    * undefined
      * 변수 값 없음을 나타내는 타입
      * 변수 선언 이후 값 할당하지 않았을 때 **자동으로 할당** / JS가 자동으로 할당
    * null
      * 변수의 값 없을 때 **의도적**으로 표현할 때 사용 / JS 자동 할당X 사용자가 의도적으로 나타냄
      * null 타입과 typeof 연산자
        * typeof -> 자료형 평가 위한 연산자 / 결과는 객체로 표현된다
    * Boolean(불리언)
      * 논리적 참 or 거짓
      * true or false로 표현함
      * 조건문이나 반복문에서 주로 사용하고, 자동 형변환 규칙이 있다.
      * JS에서는 비어있는 오브젝트나 리스트를 `True`로 반환
  * 참조타입
    * 객체 타입 자료형 / 변수에 객체 참조 값 담긴다 / 복사할 때 참조 값이 복사된다.



* 연산자
  * 할당 연산자
    * 오른쪽에 피연산자의 평가 결과 / 왼쪽 피연산자에 할당하는 연산자
      * Increment(++) : 피연산자 값을 1증가
      * Decrement(--): 피연산자 값을 1 감소
      * Airbnb Style Guide에서는 '+=', '-=' 같이 분명하게 적을 것 권장
  * 비교 연산자
    * 비교 결과값을 불리언으로 반환
    * `<`, `>`, `<=`, `>=`
  * 동등 비교 연산자(==) -> 잘 사용하지 않음
    * 두 피연산자가 값은 값으로 평가 되는지 비교하고, 불리언 값 반환한다.
    * 비교할 때 암묵적 타입변환을 통해 타입 입치 후 같은 값인지 비교 한다.
      * ex) const a = 1004 / const b = '1004' / console.log(a == b) -> true
      * const c = 0 / const False / console.log(c == d) -> true
      * console.log(a + b) -> 10041004
    * 예상하지 못한 결과 발생가능성 있어서 특별한 경우 제외하고 사용 X
  * 일치 비교 연산자(===) -> JS에서 비교할 때는 주로 이거 사용
    * 엄격한 비교, 암묵적 타입 변환이 이뤄지지 않음
      * 타입, 값 모두 같은지 확인
  * 논리연산자
    * 세가지로 구성
      * or 연산 || 연산자
      * and 연산 && 연산자
      * not 연산 ! 연산자
    * 단축 평가
      * false && true -> false
      * true || false -> true
  * 삼항 연산자
    * 세개의 피연산자 사용해서 조건 따라 값 반환
    * 가장 왼쪽 조건식이 참이면 콜론(:) 앞의 값 사용 / 아니라면 콜론(:) 뒤의값 사용
    * 삼항 연산자 결과는 변수에 할당 가능
    * 한줄로 표기하는 것 권장



### ECMAScript 6 조건문과 반복문

* 조건문

  * if

    * boolean 타입으로 변환 후 참/거짓 판단
    * `if`, `else if`, `else`
      * 조건은 `소괄호()` 안에 작성
      * 실행하려는 코드는 `중괄호{}` 안에 작성한다.
      * 블록스코프를 생성함

  * switch

    * 조건 표현식의 결과값이 

    * ```javascript
      switch(expression) {
          case 'first value': {
              // do something
              break
          }
          case 'second value': {
              // do something
              break
          }
          [default: {
           // do something
           }]
      	}
      ```

    * 

    * 표현식(expression)-> (주로 변수..)의 결과값을 이용한 조건문

    * 표현식 결과값이랑 case문의 오른쪽에 있는 값 비교

    * break / default 문은 `선택적`으로 사용 가능

    * break문 없을 때, break 문 만나거나 default문 실행할 때까지 조건문 실행함

      * 파이썬 while이랑 비슷?

    

* 반복문의 종류와 특징

  * while

    * 조건문이 참(true) 인동안 반복 시행
    * 조건은 소괄호 안에

  * for

    * ```javascript
      for ( initialization; condition; expression) {
          // do something
      }
      ```

    * 세미콜론(;)으로 구분해서 세 부분으로 구성됨

    * | initialization          | condition            | expression           |
      | ----------------------- | -------------------- | -------------------- |
      | 최초 진입 시 1번만 실행 | 매 반복 시행 전 평가 | 매 반복 시행 후 평가 |
      | 초기값                  | 조건                 | 값의 증감            |

    * 블록스코프 생성한다.

  * for...in

    * ```javascript
      for ( variable in object) {
          // do something
      }
      ```

    * {key: value} 쌍, variable -> key로 순회 돈다. / value 쓰고싶으면 values() / key, value 둘 다 하고 싶으면 entries() object.entries()

    * 주로 객체 속성 순회할 때 사용 -> python에서 딕셔너리 순회할 때

    * 배열 순회 가능한데, 인덱스 순으로 순회한다는 보장이 없어서 권장X

  * for...of

    * ```javascript
      for (variable of iterables) {
          // do something
      }
      ```

    * 실행할 코드는 중괄호 안에 작성

    * 반복 가능한(iterable) 객체 순회하면서 값 꺼낼 때 사용 (Array, map, Set, String 등등)



### ECMAScript 6 함수

* 함수
  * 참조 타입 중 하나, function 타입에 속한다.
  * 정의 방법은 2가지다.
    * 함수 선언식 (function declation)
    * 함수 표현식 (function expression)
  * 자바스크립트 함수는 일급객체(First-class citizens) / Python 도 일급객체!
    * 변수에 할당 가능
    * 함수의 매개변수로 전달 가능
    * 함수의 반환 값으로 사용이 가능하다.

* 선언식 -> 함수의 이름 선언

  * ```javascript
    function add (num1, num2) {
        return num1 + num2
    }
    
    add(2, 7)
    ```

* 표현식(expression) -> 변수 이름에 넣어서 사용하는 방식

  * 표현식 내에서 함수를 정의하는 방식이다.

    * 어떤 하나의 값으로 결정되는 코드 단위

  * 이름 없는 함수 -> 익명함수(anonymous function)

  * 익명함수는 표현식에서만 사용 가능하다.

  * ```javascript
    const sub = function (num1, num2) {
        return num1 + num2
    }
    sub(7, 2)
    
    -> return 없는 경우 undefined 반환 / return 있어야함

* default arguments(기본 인자)

  * 인자 작성 시 '=' 뒤에 작성

  * ```javascript
    const greeting = function (name = 'noName') {
        console.log(`hi ${name}`)
    }
    greeting()



* 선언식 표현식 비교

  * 함수의 타입

    * 함수도 하나의 값으로 평가된다.
    * 선언식과 표현식 함수 모두 타입은 function으로 동일하다.

  * hoisting(호이스팅) 

    *  함수 선언식

      * 선언식으로 선언한 함수는 var 정의 변수처럼 hoisting 발생

      * 함수 호출 이후 선언해도 동작된다.

      * 파이썬은 아니지만, Javascript에서는 동작이 됨.

        ```javascript
        add(2, 7)
        function add (num1, num2) {
            return num1 * num2
        }
        ```

    * 함수 표현식
      * hoisting 시 에러 발생한다.
      * 함수를 변수에 할당해서 변수로 평가되어 변수 scope 규칙 따르기 때문이다.

* Arrow function

  * `=>` 화살표 표시

  * function 키워드 생략이 가능하다. -> arrow function 사용할 때 필수

  * 함수 매개변수가 한 개 뿐이라면, `()` 도 생략가능함 -> 선택

    * 단, 매개변수가 없을 경우 `()` 는 생략 하면 안됨 / `()` or  `_	` 로 대체 가능하다.

  * 함수 코드작성 부분이 표현식 하나만 있다면 `{}`, `return`도 생략이 가능하다. -> 선택

    ```javascript
    const arrow = function (name) {
        return `hello! ${name}`
    }
    
    // 1. function 키워드 삭제 / 1 단계 기본형
    const arrow = (name) => { return `hello! ${name}` }
    
    // 2. () 생략 (매개변수 하나일 경우)
    const arrow = name => { return `hello! ${name}` }
    
    // 3. {} & return 생략
    const arrow = name => `hello! ${name}`



### ECMAScript 6 배열과 객체



* 배열(Arrays)

  * 키와 속성들을 담고 있는 참조 타입의 객체다

  * 순서 보장!!

  * **`대괄호 []`**를 사용해서 생성하고, `0`을 포함한 **`양의 정수 인덱스`**로 특정 값 접근가능 (파이썬에서는 음의 정수 사용가능했지만 Javascript는 불가능)

  * 배열 길이는 array.length 형태 / 마지막 원소는 array.length - 1로 접근

  * 메서드 목록

    * | 메서드          | 설명                                          | 비고                     |
      | --------------- | --------------------------------------------- | ------------------------ |
      | reverse         | 원본 배열 순서를 반대로 정렬한다.             |                          |
      | push & pop      | 배열 가장 뒤에 있는 요소 추가 or 제거         | 파이썬 append 비슷       |
      | unshift & shift | 배열 가장 앞에 있는 요소 추가 or 제거         |                          |
      | includes        | 배열에 특정 값 존재여부 파악 후 참/거짓 반환  |                          |
      | indexOf         | 배열에 특정 값 존재여부 파악 후 인덱스 반환   | 요소 없을 경우 -1 반환   |
      | join            | 배열 안 모든 요소를 구분자 사용해서 연결한다. | 구분자 생략 시 쉼표 기준 |

    * Array helper method 

      * 배열 순회하면서 특정한 로직 수행하는 메서드

      * 메서드 호출할 때, 인자로 `callback` 함수를 받음

        * `callback` : 어떤 함수 내부에서 실행될 목적으로 인자로 넘겨받는 함수를 뜻함

      * | 메서드  | 설명                                                | 비고         |
        | ------- | --------------------------------------------------- | ------------ |
        | forEach | 배열 각 요소에 콜백 함수를 한번씩 실행한다.         | 반환 값 없음 |
        | map     | 콜백 함수 반환 값을 요소로, 새로운 배열 반환한다    |              |
        | filter  | 콜백 함수 반환 값 중 참인 요소들로 새로운 배열 반환 |              |
        | reduce  | 콜백함수 반환 값들을 하나의 값에 누적해서 반환      |              |
        | find    | 콜백 반환 값이 참이면 해당 요소 반환                |              |
        | some    | 배열 요소 중 하나라도 특정 조건 통과하면 참 반환    |              |
        | every   | 배열 요소 중 모든 요소가 특정 조건 통과하면 참 반환 |              |

        

      * forEach

        * ```javascript
          array.forEach(element, index, array) => {
              // do something
          })
          ```

        * ```javascript
          const country = ['대구', '구미', '서울', '대전']
          country.forEach((region))

        * 배열 각 요소에 대해 콜백 함수 한 번씩 실행

        * 3가지 매개변수 구성되어 있다.

          * element : 배열 요소 / 필수로 받는다.
          * index : 인덱스
          * array : 배열 자체

        * 반환 값이 없다.

        

      * map

        * forEach와 동일하게 콜백 함수 한 번씩 실행

        * ```javascript
          array.map((element, index, array) => {
              // do something
          })
          ```

        * 대신, 콜백 함수 반환 값 요소로 새로운 배열 만듦

        * ```javascript
          const arr = [9, 8, 7, 6, 5]
          
          const arrDouble = arr.map((arrr) => {
              return arrr * 2
          })
          console.log(arrDouble)  // [18, 16, 14, 12, 10]
          ```

        

      * filter

        * 형태는 위의 애들과 동일함

        * ```javascript
          array.filter((element, index, array) => {
              // do something
          })
          ```

        * 대신, 콜백 반환 값이 참인 것들만 모아서 새로운 배열 만든다.

        * ```javascript
          const arr = [9, 8, 7, 6, 5]
          
          const arrOdd = arr.filter((arrr, index) => {
              return arrr % 2
          })
          console.log(arrOdd)  // [9, 7, 5]
          ```

        

      * reduce

        * 형태 비슷한데, 제일 앞에 `acc` 들어감, 그리고 마지막에 `initialValue` 들어감

        * ```javascript
          array.reduce((acc, element, index, array) => {
              // do something
          }, initialValue)
          ```

        * 콜백 함수 반환 값들 `acc`에 누적해서 반환한다.

        * `acc` : 과거의 결과를 담아둔다.

        * `initialValue` : `acc`의 초기값이다. / 설정 안하고 비워두면, 배열 첫 번째 값 자동으로 사용

        * ```javascript
          const arr = [9, 8, 7]
          
          const result = arr.reduce((acc, arrr) => {
              return acc + arrr
          }, 0)
          console.log(result)  // 24
          ```

          

      * find

        * 검색하는거 / 반환 값이 참이면 해당 요소 반환 / 없으면 undefined 반환

        * ```javascript
          array.find((element, index, array) => {
              // do something
          })
          ```

        * ```javascript
          const
          ```

        

      * some

        * 배열 안에서 하나라도 콜백함수 통과하면 참 반환한다.

        * 배열이 비어있으면 항상 거짓(false) 반환한다.

        * ```javascript
          array.some((element, index, array) => {
              // do something
          })
          ```

          

      * every

        * 배열 안 모든 값이 콜백 함수 통과해야 참 반환

        * ```javascript
          array.every((element, index, array) => {
              // do something
          })

* 객체(objects)

  * 객체는 속성의 집합, 중괄호 내부에 key, value 쌍으로 표현한다.

  * key는 문자열 타입만 가능하다!!

  * value는 모든 타입 가능!

    

* 객체관련 문법

  * shorthand
    * 속성명 축약
      * 객체 정의 시 key와 할당하는 변수 이름 같으면 축약 가능하다.
    * 메서드명 축약
      * 메서드 선언 하면 function 키워드 생략 가능하다.
  * 계산된 속성
    * key의 이름을 표현식 사용해서 동적으로 생성 가능하다!
    * 참고 : https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Operators/Object_initializer
  * 구조 분해 할당 (destructing assignment)
    * 배열 또는 객체 분해하여 속성을 변수에 쉽게 할당 할 수 있다.

  

* JSON

  * key-value 쌍 형태로 데이터 표기하는 언어 독립적 표준 포맷
