> JavaScript



* JavaScript
  * 브라우저 화면은 '동적'으로 만들 수 있다.
  * 브라우저를 조작할 수 있는 유일한 언어이다.
  * HTML이 뼈대, CSS는 살을 붙인거라면, JavaScript는 움직일 수 있게 행동가능하게 만들어준다.
  * 한 때, 각기 다른 모습의 자바스크립트가 만들어지면서 브라우저 간 호환이슈가 발생되어 표준화가 진행되었다. 
  * 1997년 ECMAScript1 (ES1)이 탄생하고, 2015년 ES2015 (ES6) 탄생하면서, 이후엔 통상 ES6라 부르며 표준이 넘어온 상태 



* browser(브라우저)

  * HTML 파일이나 문서를 출력하는 GUI 기반의 소프트웨어
  * 주요 브라우저로는 Google Chrome, Firefox, Microsoft Edge.....

  * 브라우저에서 가능한 일

    * DOM 조작은 문서(HTML)을 조작하는 것
    * BOM 조작 -> XHR / location / screen / navigator / frames / history
    * JavaScript Core (ECMAScript) -> Iteration / Data Structure ... 

    

  * DOM (Document Object Model)

    * HTML 조작 ex) windows.document.title = 'JavaScipt'

    * 문서를 구조화함. 구조화된 요소를 하나의 객체로 취급해서 다루는 논리적 트리모델

    * 구조화할 때 해석하는 과정이 필요한데, 이를 Parsing(파싱) 이라고 함

      * 브라우저가 문자열 해석해서 DOM Tree로 만들어감

    * 조작

      * 선택 하고 변경한다. 순서 중요! 선택 필수

      * 선택 관련한 메서드

        * 반환 타입

          | 단일 element     | HTMLCollection(live collection) | NodeList(static collection) |
          | ---------------- | ------------------------------- | --------------------------- |
          | getElementById() | getElementByTagName()           | document.querySelectorAll() |
          | querySelector    | getElementByClassName()         |                             |

          * HTMLCollection

            * id, name, 인덱스 속성으로 각 항목에 접근 가능

          * NodeList

            * 인덱스 번호로만 접근 가능

          * 둘 다 Live collection으로 DOM 변경사항 실시간 반영하지만, NodeList는 Static collection

            | Live Collection         | Static Collection (non-live) |
            | ----------------------- | ---------------------------- |
            | 바뀔 때 실시간 업데이트 | 변경되어도 영향 주지 않음    |

          

        * document.querySelector() : 선택자와 일치하는 엘리먼트 하나 선택 반환 / 없다면 null 반환

        * document.querySelectorAll() : 선택자와 일치하는 여러 엘리먼트 선택 / NodeList를 반환

        * getElementByTagName() / ID() / ClassName()

      * 변경 관련한 메서드

        * document.createElement() : 태그명을 사용해서 HTML 요소 반환

        * 부모노드.append() : 마지막 자식 노드 다음에 삽입, 여러개 추가 가능

        * 노드.appenChilde() : 한 노드를 특정 부모 노드에 마지막으로 삽입

        * ChildeNode.remove() : 트리로 부터 특정한 객체 제거함

        * Node.removeChild() : 자식 노드 제거하고 제거한 노드 반환

        * 속성

          * Node.textContent : 노드의 모든 요소 가져옴 

            ex) `<h1> 제목 <h1>` 제목크기가 아닌 태그까지 다 나옴

          * Node.innerText : 렌더링 된 모습 표현, 스타일링 적용된 최종 모습이 표현됨

            ex) `<b>찐하게` -> **찐하게** / 렌더링 된 모습

          * Element.innerHTML : 엘리먼트 포함된 HTML 반환 / XSS 취약점

          * Element.setAttribute(name, value) : 지정한 요소 값 설정함 / 값 존재 시 업데이트, 아니면 새로 추가함

          * Element.getAttribute() : 해당 요소 값 가져옴

      * XSS ( Cross-site scripting)

        * 공격자가 클라이언트 측에 악성스크립트 심어서 공격. 실행은 피해자가 해서, 피해자가 공격자가 됨

  * BOM (Browser Object Model)

    * ex) location.host -> "www.naver.com" / window.print() <인쇄> / window.open() <탭 창>
    * 브라우저와 자바스크립트가 소통하기 위한 모델임
    * 브라우저 프레임, 창을 추상화해서 프로그래밍적으로 제어할 수 있도록 해주는 수단

    

  * Event

    * 활동, 사용자와 상호작용 같은 발생을 알리기 위한 객체
    * 마우스 클릭이나 이동, 키보드 입력 등 행동이나 메서드 호출해서 프로그래밍적으로도 만들 수 있음
    * 어떤 ~ 행동을 하면 ~ 결과를 준다.
    * EventTarget.addEventListener(type, listener)
      * type : 반응할 유형( ex) click, input ...)
      * listener : 이벤트 발생했을 때 반환할 행동에 알림 받는 객체
